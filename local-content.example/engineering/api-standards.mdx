# API Design Standards

All internal and external APIs must follow these conventions to ensure consistency and predictability.

## URL structure

```
https://api.teamssquared.com/{version}/{resource}/{id?}/{sub-resource?}

# Examples
GET  /v1/users
GET  /v1/users/usr_abc123
GET  /v1/users/usr_abc123/projects
POST /v1/projects
```

- Use **kebab-case** for multi-word resource names: `/api-keys` not `/apiKeys`
- Always version at the path level (`/v1/`, `/v2/`)
- IDs use typed prefixes: `usr_`, `proj_`, `doc_`

## HTTP methods

| Method | Use |
|--------|-----|
| `GET` | Read resource(s), never mutate |
| `POST` | Create a new resource |
| `PUT` | Full replacement of a resource |
| `PATCH` | Partial update |
| `DELETE` | Remove a resource |

## Request and response format

All bodies are **JSON**. Always set `Content-Type: application/json`.

```typescript
// Standard success envelope
interface ApiResponse<T> {
  data: T;
  meta?: {
    total?: number;
    page?: number;
    perPage?: number;
  };
}

// Standard error envelope
interface ApiError {
  error: {
    code: string;       // machine-readable, e.g. "RESOURCE_NOT_FOUND"
    message: string;    // human-readable
    details?: unknown;  // optional validation errors etc.
  };
}
```

## Error codes

```typescript
// Use these HTTP status codes consistently
const STATUS = {
  OK:                  200,
  CREATED:             201,
  NO_CONTENT:          204,
  BAD_REQUEST:         400,
  UNAUTHORIZED:        401,  // not authenticated
  FORBIDDEN:           403,  // authenticated but not authorised
  NOT_FOUND:           404,
  CONFLICT:            409,
  UNPROCESSABLE:       422,
  TOO_MANY_REQUESTS:   429,
  INTERNAL_ERROR:      500,
} as const;
```

## Authentication

All endpoints (except `/health`) require a Bearer token:

```http
GET /v1/users HTTP/1.1
Authorization: Bearer eyJhbGciOiJSUzI1NiJ9...
```

Token validation is handled by the `@ts2/auth` middleware — do not implement it yourself:

```typescript
import { requireAuth } from "@ts2/auth";

router.get("/v1/users", requireAuth(), async (req, res) => {
  // req.user is guaranteed to be set here
  const users = await getUsers({ requesterId: req.user.id });
  res.json({ data: users });
});
```

## Pagination

Use cursor-based pagination for lists:

```typescript
// Request
GET /v1/projects?cursor=proj_xyz&limit=20

// Response
{
  "data": [...],
  "meta": {
    "nextCursor": "proj_abc",
    "hasMore": true
  }
}
```
