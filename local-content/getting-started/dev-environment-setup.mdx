# Dev Environment Setup

Follow these steps to set up your local development environment. Estimated time: ~45 minutes.

## Prerequisites

- macOS 13+ or Ubuntu 22.04+ (Windows via WSL2 is supported)
- Admin access to your machine

## 1. Install core tools

```bash
# macOS — install Homebrew first
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Then install the core stack
brew install node@22 git gh pnpm
brew install --cask docker
```

```bash
# Ubuntu / WSL2
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs git docker.io
npm install -g pnpm
```

## 2. Clone the monorepo

```bash
gh auth login          # authenticate with GitHub
gh repo clone teams-squared/monorepo
cd monorepo
pnpm install           # install all workspace dependencies
```

## 3. Configure environment variables

Each service has an `.env.example`. Copy and fill in the values:

```bash
# from the repo root
for dir in apps/*/; do
  if [ -f "$dir/.env.example" ]; then
    cp "$dir/.env.example" "$dir/.env.local"
    echo "Created $dir/.env.local"
  fi
done
```

> **Note:** Real secrets are in 1Password under the **Engineering > Local Dev** vault. Ask your onboarding buddy for access.

## 4. Start the development stack

```bash
pnpm dev          # starts all services via Turborepo
```

Services start at:
- Web app: http://localhost:3000
- API: http://localhost:4000
- Docs portal: http://localhost:3001

## 5. Verify setup

```typescript
// Run this sanity check from the repo root
import { healthCheck } from "@ts2/core";

const result = await healthCheck();
console.log(result); // { status: "ok", services: [...] }
```

Or via the CLI:

```bash
pnpm ts2 health
# ✓ database     connected (12ms)
# ✓ redis        connected (3ms)
# ✓ api          reachable (45ms)
```

## Troubleshooting

### Port already in use

```bash
# Find what's using a port
lsof -i :3000

# Kill it
kill -9 $(lsof -t -i:3000)
```

### Docker not starting

Make sure Docker Desktop is running before running `pnpm dev`. On Linux:

```bash
sudo systemctl start docker
sudo usermod -aG docker $USER   # run once, then log out and back in
```
