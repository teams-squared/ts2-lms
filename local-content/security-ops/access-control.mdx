# Access Control Procedures

This document is **restricted to administrators**. Do not share with employees unless access has been explicitly approved by the CISO.

## Principles

Access at Teams Squared follows the principle of **least privilege**: every account is granted only the minimum access required to perform its function.

- No shared credentials
- All privileged access via SSO + MFA
- Access reviewed quarterly; unused accounts disabled within 30 days

## Granting access

### Standard employee access

Standard access is provisioned automatically via Okta when an employee is created in BambooHR. No manual action is needed.

If auto-provisioning fails:

```bash
# Trigger manual provision via CLI
ts2-admin provision-user \
  --email alice@teamssquared.com \
  --groups engineering,docs-portal \
  --dry-run   # remove to apply
```

### Elevated / admin access

Elevated access requires a written request approved by the employee's VP and the Security team:

1. Employee submits request in Linear (team: **Security**, label: **access-request**)
2. VP approves in Linear within 5 business days
3. Security team provisions and records in the access log
4. Access is time-limited: 90 days by default, renewable with re-approval

```typescript
// Internal provisioning API — admins only
const result = await adminClient.grantRole({
  userId: "usr_abc123",
  role: "admin",
  expiresAt: addDays(new Date(), 90),
  approvedBy: "vp-engineering@teamssquared.com",
  ticketId: "SEC-441",
});
```

## Revoking access

Revoke access immediately when:
- An employee is offboarded
- A contractor engagement ends
- A security incident involves the account

```bash
ts2-admin revoke-user --email contractor@example.com --reason offboarding
# Revokes Okta, GitHub, AWS, and database access simultaneously
```

> **SLA:** Offboarding requests must be completed within **2 hours** of HR notification.

## Quarterly access review

Every quarter, the Security team runs:

```bash
ts2-admin access-report --format=csv > access-review-$(date +%Y-Q%q).csv
```

Each manager receives a list of their direct reports' access. They must confirm or revoke within 2 weeks. Unreviewed accounts are automatically suspended.

## Audit log

All access changes are written to the immutable audit log in CloudWatch:

```
arn:aws:logs:eu-west-1:123456789:log-group:/ts2/access-audit
```

Retention: 7 years (compliance requirement).
